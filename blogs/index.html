1st February 2021

Third Blog

The third week of semester two, we started working on the LED Matrix.

We connected the LED Matrix onto the Arduino board using female to male wires and crocodiles clips into the Appropriate pins with the aid of the datasheet provided for the MAX7219 LED Driver.

The VBATT pin from the Arduino is connected to the VCC of the LED which is used to power the LED, The GND is connected to the GND, The DIN is connected to the #6 Pin in the Arduino board which is responsible for the interpreting the code that is inputted, The CS is conntected to the #12 Pin which is responsible for the input delay. The CLK Pin is connected to the #10 Pin.

We completed our prototype , which was one of the most basic programs we could produce. The prototype basically printed "Hello World" on the LED matrix , which can be seen in the video below.

Near the end of week three we created a program to turn on certain led lights on the adafruit playground classic depending on sound volume.

Achieving this , helped us figure out how to use the microphone on the playground circuit and display the LED lights on the LED matrix depedning on volume.

By the end of week three , our project was capable of playing animations on a 8x8 matrix with sound.

Fourth Blog

The fourth week of semester two, our objective was to play animations on 8x32 matrix.

Which we completed at the beginning of the week , even though our project was able to display animations with sound. We were unhappy as the mouth animation would not stay open for how long the user was speaking.

We modified the code by converting the microphones audio input from "baud" to "decibel".As we were more educated on the decibel unit of measurement than baud , this allowed us to make the mouth animation more realistic.Now the mouth animation would stay open depending on how long the user would be speaking for.

The issues we still face is the accelerometer loop seems to be infinite,user may need to take quite long pauses when speaing into the microphone for a realistic mouth movement and finally working on differnt set of word patterns to mimic a mouth.

22nd February 2021

Sixth Blog

In the sixth week we tried figuring out a way to pattern match by using a counter that measures how long the user will speak and another counter for counting the decibels. The thought process of this was that when a user speaks "Hello" and while checking the serial monitor the dB value ranges from 60 - 63 and appears 3 times in the serial which is the duration of how long it took to say the sentence. Conditions would be then placed to check for these values and display the corresponding animation

Counter for duration of speaking

Matthew added Test 6 into the testing directory which explains the process of using the accelerometer mentioned in last weeks blog

Test 6

We have been testing each of the components individually up to now and are working properly. A problem arised when you try to make all the components work together at the same time, the accelerometer inhibits the MAX7219 LED from lightning when initializing the library for the CircuitPlayground. We asked our supervisor what might be the problem and advided us that it might be problems within the SPI bus. This was the problem as when checking the libraries (CircuitPlayground.h and LEDcontrol.h) the SPI connections were conflicting. Numerous hours was spent trying to figure out how to create new connections for the SPI but failed and decided to buy a different LED Matrix which had less Pins needed to run it. The WS218 LED panel was the solution for this as it only needed 3 Pins DIN, 5V and GND. A new library was used and a different way of mapping animations into the LED panel. The new panel was also smaller and more appropriate to be put in a mask and allowed for colors to change which the other did not.

Components all working together without issue
